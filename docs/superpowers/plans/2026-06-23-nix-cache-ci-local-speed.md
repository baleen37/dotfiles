# Nix Cache CI Local Speed Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make CI reliably populate the shared Nix binary cache so local `nix build` and `make switch` can substitute existing paths instead of rebuilding them.

**Architecture:** Use Cachix `baleen-nix` as the primary cross-machine binary cache because it benefits both CI and local machines. Keep GitHub `actions/cache` as an optional CI-only helper, and only save it after the Nix workload has produced cacheable data. Preserve the existing centralized cache settings in `lib/cache-config.nix`, `flake.nix`, `.github/workflows/ci.yml`, and `.github/actions/setup-nix/action.yml`.

**Tech Stack:** Nix flakes, nix-darwin, NixOS, Cachix, GitHub Actions, `actions/cache`, Nix unit checks.

- [ ] **Step 2: Add save step to CI after the Nix workload**

In `.github/workflows/ci.yml`, add this step after `Upload to Cachix`:

```yaml
- name: Save Nix cache
  if: always()
  uses: actions/cache/save@0057852bfaa89a56745cba8c7296529d2fc39830 # v4.3.0
  continue-on-error: true
  with:
    path: |
      ~/.cache/nix
      ~/.local/state/nix
    key: nix-${{ runner.os }}-${{ runner.arch }}-${{ hashFiles('**/flake.lock') }}-v4-${{ github.run_id }}-${{ github.run_attempt }}
```

Use the run-specific suffix because `actions/cache/save` cannot overwrite an existing cache key. The existing restore prefix `nix-${{ runner.os }}-${{ runner.arch }}-` can still pick up the newest matching cache.

Also update both restore steps in `.github/actions/setup-nix/action.yml` so same-lockfile caches are preferred before the broad runner-level fallback:

```yaml
restore-keys: |
  nix-${{ runner.os }}-${{ runner.arch }}-${{ hashFiles('**/flake.lock') }}-v4-
  nix-${{ runner.os }}-${{ runner.arch }}-
```

- [ ] **Step 1: Add README note**

In `README.md`, near the existing Nix/Determinate cache setup section, add:

```markdown
### Nix cache behavior

Local builds are accelerated by binary substituters, especially `https://baleen-nix.cachix.org`, `https://nix-community.cachix.org`, and `https://cache.nixos.org/`. When CI pushes a built closure to Cachix on `main` or tags, local `nix build` and `make switch` can download matching store paths instead of rebuilding them.

GitHub `actions/cache` is CI-only. It can speed repeated workflow runs on GitHub-hosted runners, but it does not make local macOS builds faster. Do not store credentials in cached paths.
```
